# Security Policy

## Supported Versions

Currently, only the latest major release of Astra is actively supported with security updates. 

| Version | Supported          |
| ------- | ------------------ |
| 4.0.0   | :white_check_mark: |
| < 4.0.0 | :x:                |

## Reporting a Vulnerability

If you discover a vulnerability in Astra, please do **NOT** open a public issue. Providing a public outline of an exploit may put servers running Astra at risk.

Instead, please report vulnerabilities by contacting the developer directly:
1. Join the official [Support Server](https://discord.gg/NZ5Gr7eqE8).
2. Open a direct message or a private ticket with `watispro1`.
3. Provide details of the vulnerability, steps to reproduce, and any potential mitigations.

We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress.

### Scope
- **In Scope:** Discord API rate limit bypasses, permission elevation within the bot commands, database injection or data leaks, arbitrary code execution.
- **Out of Scope:** Volumetric DDoS attacks against the hosting server, issues requiring physical access to the host machine.
