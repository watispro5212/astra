Quick Start

Deploying Astra to your sector is a streamlined process. Once the bot is present, use the following diagnostic commands to verify system integrity.

/system status — View core engine diagnostics
/info stats — Global sector telemetry

The **Titan Engine** will automatically synchronize its tactical assets upon arrival. If commands do not appear, follow the synchronization protocol below.

Neural Sentinel

The **AI Sentinel** is a high-availability neural processing module that operates exclusively within secure **Direct Message (DM)** sectors. It features a **Triple-Key Redundant Array** with automated **Spectral Cycling** failover.

API Key Acquisition

To maintain operational integrity, Astra requires valid Google Gemini API keys. If your current nodes are reporting EXHAUSTED or QUOTA_EXCEEDED, follow these steps:

1. Navigate to Google AI Studio
2. Generate a new API Key (Gemini 1.5 Flash recommended)
3. Update the keys in your .env configuration

Tactical Commands

/ai model — Switch between speed and analysis modes
/ai status — Diagnostic report for all neural nodes
/ai reset — Clear current conversational context

Operatives can recalibrate the Sentinel's neural matrix (change models) using the /ai model command. Available models include Titan Flash (Speed) and Titan Pro (Deep Analysis).

Prefix Interface

While Astra utilizes Slash Commands for primary protocols, a traditional **Prefix Interface** is active for rapid tactical input. The default operational prefix is -.

Default Prefix: -
Usage: -[command] [options]

You can configure the sector-specific prefix using the /config prefix command (Administrator clearance required).

Command Synchronization

Due to Discord's global caching, some protocols (like /economy harvest) may experience indexing delays. If a command is missing from your terminal:

1. Type /system sync
2. Wait for "Synchronization Successful"
3. Restart your Discord terminal (Client)

This forces a nuclear purge of the sector's command cache and re-deploys the latest **Titan v7.5.0** assets.

Cloud Infrastructure

Astra has migrated from legacy SQLite to **Supabase Cloud (PostgreSQL)**. This ensures permanent data persistence even across rapid deployment cycles.

• High-Throughput Pooling active
• Automated Redundancy enabled
• Permanent Asset Persistence active

Industrial Finance

Astra's fiscal ecosystem is divided into three primary sectors: Extraction, Industry, and Speculation.

/economy mine — High-risk resource extraction
/economy harvest — Passive industrial yield collection
/stockmarket market — Live speculation feed

Operatives should utilize /economy harvest to secure yields from their industrial infrastructure. Yields are calculated based on your total operational capacity. Note: If /economy harvest is not visible, use /system sync.
