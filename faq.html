ARCHIVAL FAQ

Instant intelligence for common questions about the Astra Nova v7.0.0 engine.

General
Q What is Astra Nova v7.0.0? ▼
Astra is a free, enterprise-grade Discord bot with 9 tactical sectors covering moderation, leveling, economy, support tickets, giveaways, welcome systems, utilities, info, and core system operations. v7.0.0 "Nova" is the largest release in Astra's history, adding 3 entirely new sectors and dozens of new commands.
Q Is Astra free to use? ▼
Yes — 100%. All Patreon paywalls were removed in v6.1.0 and every feature in Nova v7.0.0 is available to every server at no cost. You can self-host under the Astra Community License v2.1 as well.
Q How do I add Astra to my server? ▼
Click the "Establish Link" button on the homepage. Astra requests Administrator permissions to ensure it can manage channels, roles, and messages across all sectors. Once added, slash commands are available immediately — no manual setup required.
Moderation
Q How does the warning system work in v7.0? ▼
Use /mod warn @user reason. The warning is logged as a moderation case in the database and an embed is DM'd to the target operative. If their DMs are closed, the warning is still recorded and staff are notified. You can view all warnings with /mod history @user.
Q Can I view a specific moderation case? ▼
Yes. Use /mod case [number] to retrieve full details on any case: type, reason, moderator, duration, timestamp, and status. Use /mod history @user to pull up to 15 most recent cases for a specific operative.
Economy & Leveling
Q Is the leveling system global or per-server? ▼
Global. Astra uses a single XP database across all linked communities. Your level and XP carry over between servers, so your rank is preserved no matter which Astra-linked community you're in.
Q What are the economy income sources in v7? ▼
There are three income streams: /economy daily (2,500 credits, 24h cooldown), /economy work (250–750 credits, 1-hour cooldown), and /economy mine (high-risk: 70% chance of 500–2,500 credits, 30% chance of losing up to 1,000 credits, 30-min cooldown).
Tickets & Giveaways
Q How do I set up the ticket system? ▼
The ticket system works out-of-the-box with no setup. Members run /ticket create and a private channel is created. For advanced configuration (category placement, automatic staff role access), an admin can configure the ticket settings in the database using the ticket_configs table — or wait for the upcoming /ticketconfig command in a future update.
Q What happens when a giveaway timer expires? ▼
The Giveaway Service background checker runs every 60 seconds. When a giveaway's ends_at timestamp passes, it automatically fetches 🎉 reactions, filters out bots, picks random winners, announces them in the channel, and marks the giveaway as ended. No manual intervention needed.
Welcome & Setup
Q How do I configure the welcome message? ▼
Use /welcome set-channel #channel to set where messages appear, /welcome set-message your text to write the message (use {user} and {server} as placeholders), and /welcome set-role @role to auto-assign a role on join. Run /welcome test to preview it instantly. Use /welcome view to see the full current config.
Q How does the Vanguard Error Reporter work? ▼
Whenever a command throws an unhandled exception, Astra generates a diagnostic packet (command name, guild, user, error message) and DMs it to the configured owner ID. This allows near-instant identification and resolution of production failures without users needing to report them manually.
Q What does the Apex Nuclear Protocol do? ▼
Running /setup_server with nuclear mode enabled completely purges all channels and roles in the server, then rebuilds the full Apex hierarchy from scratch — including 13 custom roles and 7 categorised sectors (Information, Citizen, Elite, Security, Logistics, Economy, Archive, and Voice). This is irreversible and intended for initial server configuration only.
