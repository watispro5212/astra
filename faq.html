Frequently Asked Questions

Quick answers to the questions we hear most.

Click the "Invite Astra" button on this website. You'll be taken to Discord's OAuth2 authorization page where you can select the server you want to add Astra to. Make sure you have the Manage Server permission in the target server.
Astra works best with these permissions: Send Messages, Embed Links, Manage Messages, Kick Members, Ban Members, Manage Roles, Manage Channels, and Read Message History. You can customize permissions based on which features you want to use. For example, if you don't need moderation, you can skip Kick/Ban permissions.
No. Astra uses Discord's native slash command system exclusively. Type / in any channel where Astra has access and you'll see all available commands in the autocomplete menu. This provides better discoverability, validation, and permission control.
Astra stores only what's needed to provide its features: server configuration settings (channel IDs, thresholds), moderation cases (user IDs, reasons, timestamps), poll data (votes), reminder data (message, trigger time), starboard entries, ticket configurations, and role menu settings. Messages content is not stored. See our Privacy Policy for full details.
Yes — completely free. All features are available to every server with no premium tiers, subscriptions, or paywalls.
Yes. Astra supports multi-server operation. Each server has its own independent configuration, moderation cases, and settings. Changes in one server never affect another.
Use /config starboard #your-channel to set the starboard channel, then /config threshold 3 to set how many ⭐ reactions a message needs. Once set, Astra will automatically repost qualifying messages. Stars from bots and the message author are excluded from the count.
First, run /ticket_setup and select the category where tickets should be created and the staff role that gets access. Then run /ticket_panel #channel to send the ticket launcher button to a channel. Users click the button to open a private support ticket.
Join our support server and contact a team member. We can remove all stored data associated with your server or user ID upon request. You can also remove Astra from your server, which will stop all data collection — though existing records will remain until a deletion request is made.
Astra's source code is available on GitHub. Contributions are welcome — feel free to submit issues, feature requests, or pull requests. Self-hosting is also possible if you prefer to run your own instance.
